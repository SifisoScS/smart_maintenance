## ğŸ§  **Design Patterns Used**

| Pattern                       | Purpose                                                               |
| ----------------------------- | --------------------------------------------------------------------- |
| **Factory**                   | Create specialized maintenance requests (Electrical, Plumbing, HVAC). |
| **Repository**                | Abstract database operations for each entity (UserRepo, RequestRepo). |
| **Service Layer**             | Encapsulate business logic separate from controllers.                 |
| **Observer**                  | Notify technicians/admins when maintenance requests change status.    |
| **Strategy**                  | Allow different notification strategies (Email, In-App).              |
| **Singleton**                 | Central configuration or database connection management.              |
| **Dependency Injection (DI)** | Pass repositories/services to controllers for flexibility.            |
| *(Optional)* **Decorator**    | Add logging or caching to service methods.                            |
